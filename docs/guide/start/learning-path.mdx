# Dora 新手学习路线

如果你第一次听说 Dora，不建议一上来就翻完整 API。更稳的路径是：先知道它解决什么问题，再跑通一个最小数据流，最后带着问题去读概念和示例。

:::tip 适合谁
这条路线面向第一次接触 Dora 的中文开发者、具身智能/机器人应用开发者，以及正在从 ROS、Python AI demo 或多进程脚本迁移到更模块化架构的人。
:::

## 1. 先建立整体印象

先读：[介绍：DORA 是什么？](/guide/start/introduction)

你需要先理解三件事：

- Dora 把机器人应用组织成一个数据流图。
- 图里的每个节点只负责一件事，例如采集图像、运行模型、显示结果。
- 节点之间通过消息和事件流连接，而不是把所有逻辑写在一个脚本里。

读完这一页后，不需要立刻理解所有内部机制。你只要能回答“Dora 为什么适合拆分机器人应用”就够了。

## 2. 跑通命令行工具

下一步读：[快速上手](/guide/start/quick-start)

目标不是背命令，而是确认本地环境能跑：

```bash
dora -h
```

如果这一步失败，先解决安装问题。不要跳到复杂示例，否则后面排错会很痛苦。

## 3. 理解最小数据流

跑通 CLI 后，重点看数据流文件里的几个概念：

- `nodes`：应用里的节点列表
- `inputs`：这个节点接收哪些消息
- `outputs`：这个节点产出哪些消息
- 节点 ID：其他节点如何引用它

你可以把 dataflow 看成机器人应用的“接线图”。先看懂接线图，再看每个节点内部代码。

## 4. 再读基础主题

建议按这个顺序读：

1. [数据流](/guide/topics/dataflow)：理解 Dora 的核心组织方式
2. [节点](/guide/tutorial/node)：理解节点如何处理输入和输出
3. [事件流](/guide/tutorial/event-stream)：理解消息如何在系统里流动
4. [Dora CLI](/guide/tutorial/dora-cli)：熟悉常用命令

读文档时可以带着一个问题：如果我要把“摄像头、模型推理、结果显示”拆开，哪些部分应该是节点，哪些部分应该是数据流配置？

## 5. 选择一个小示例复现

不要一开始就做完整机器人项目。更推荐从一个很小的场景开始：

- 摄像头输入 → 图像处理 → 可视化
- 模拟数据 → Python 节点 → 打印输出
- 一个 AI 模型节点 → 下游消费结果

先让数据真的流起来，再逐步替换其中一个节点。

## 6. 遇到问题去哪里问

中文入口：[中文社区介绍](/doracc/)

建议提问时带上：

- 操作系统和 Dora 版本
- 你运行的命令
- dataflow 配置
- 报错全文
- 你已经尝试过的排查步骤

这样社区成员更容易判断问题出在安装、配置、依赖还是节点代码。
