# 1.4 四种通信模式（先认个脸）

节点之间在黑板上"交流"，其实有**四种不同的方式**。这一节我们只**认个脸、记个大概**，具体怎么用，留到[第六章](../communication/)专门展开。

还是用课堂的画面，一句话记住每一种：

| 方式 | 课堂类比 | 一句话 |
|------|---------|--------|
| **主题（Topic）** | 老师站上讲台喊："大家看黑板！" | 一个发、多个收，发完不管 |
| **服务（Service）** | 小明问小红："第三题答案？"小红答"42" | 一问一答，有来有回 |
| **动作（Action）** | 老师布置一张卷子，同学边做边举手报进度 | 长任务，边做边反馈 |
| **流式（Streaming）** | 老师带读课文，你随时可举手打断 | 连续不断，可中途打断 |

:::info 小莫说
现在不用背这些！你只要知道"我和零件们有四种聊天方式"就够啦。等我真正需要它们时，第六章会手把手教。
:::

## 为什么现在先不展开

对零基础来说，一次性塞四种模式容易懵。我们的策略是：

- **这一章**：只建立"有四种方式"的印象；
- **后面章节**：真正用到哪种，就在那里讲透。

其中最常用、最先接触的是 **Topic（主题）**——下一门项目（小飞机）里，"控制器→小飞机"的指令，走的就是它。

## 小结

- DORA 有四种通信方式：**Topic、Service、Action、Streaming**。
- 分别对应：广播喊话、一问一答、长任务边做边报、连续可打断。
- 现在记个脸即可，详解见[第六章](../communication/)。
