# 第六章 · 节点如何对话：四种通信模式

> 本章小多将解锁：**💬 掌握 4 种沟通方式**

前几章里，节点之间已经在"说话"了，但只用了最基础的一种方式。这一章，我们把 DORA 的**四种通信模式**一次讲透——让小多学会广播、问答、派任务、连续对话等各种"沟通姿势"。

## 本章你会学到

- **[6.1 Topic 发布订阅](./topic)**：一发多收、发完不管，最基础的通信方式。
- **[6.2 Service 请求应答](./service)**：一问一答，用 `request_id` 配对。
- **[6.3 Action 长任务](./action)**：边做边报进度、可取消的耗时任务。
- **[6.4 Streaming 流式](./streaming)**：连续可打断的数据流，语音视频的基础。
- **[🎯 小项目③：问答 / 任务交互](./project-interaction)**：把多种模式组合成交互中枢。

## 学完能做什么

- 分清四种通信模式各自适合什么场景；
- 用元数据（metadata）实现请求配对、任务状态、流式重组；
- 理解"四种模式底层都是发布订阅"这一核心设计。

- 记得第一章[四种通信模式先认个脸](../concepts/comm-patterns-intro)。
