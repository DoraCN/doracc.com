# 第四章 · Python 节点开发基础

> 本章小多将解锁：**✍️ 会表达：亲手写节点**

前三章，小多身上的零件都是别人写好的，你只是运行它们。从这一章开始，**你要当零件设计师**——亲手用 Python 写出小多的第一个节点，让它真正"会表达"。

## 本章你会学到

- **[4.1 节点生命周期](./node-lifecycle)**：搞懂一个节点从出生到退场的"三段式"一生，理解事件循环。
- **[4.2 写一个 Python 节点](./write-node)**：亲手写出发送者和回声两个节点。
- **[4.3 把节点接进数据流](./connect-node)**：用 `dataflow.yml` 把节点串起来，跑通你的第一条数据流。
- **[4.4 定时器与参数](./timer-params)**：让节点按节奏干活，并用参数让同一份代码更通用。

## 学完能做什么

- 独立写出一个能收发数据的 DORA Python 节点；
- 用 YAML 把多个节点连成一条数据流并运行；
- 用定时器控制节奏、用环境变量给节点传参数。

## 前置要求

- 读完[第一章](../concepts/)，理解"黑板与课堂"比喻；
- 装好[朵拉魔盒](../environment/)、跑通过[第三章小飞机](../first-dataflow/)；
- 会一点点 Python，不熟先看[附录 · Python 极简速成](../appendix/python-crash-course)。
