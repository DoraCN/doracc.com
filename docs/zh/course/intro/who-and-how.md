# 适合谁 · 怎么学

## 这门课适合谁

- 对**机器人**和**人工智能**感兴趣，想亲手实践的你；
- 有 **Linux / 命令行**基础，能输命令的开发者；
- 学过一点编程、但没接触过机器人的爱好者；
- 听说过 ROS，想了解更快、更简单的新方式的人。

## 你**不**需要这些

- ❌ 不需要昂贵的**独立显卡（GPU）**——普通电脑的 CPU 就够。
- ❌ 不需要购买**真实机器人硬件**——我们用仿真，人人可跟做。
- ❌ 不需要 **ROS/ROS2 经验**——我们会用最直白的比喻从头讲。
- ❌ 不需要提前会 **Python**——会其他编程语言即可快速上手。

## 你需要准备的

- 一台电脑：**Windows、macOS 或 Linux 都行**；
- 会使用终端（命令行）；
- 基本的编程概念（变量、函数、循环——不限语言）。

## 怎么学最轻松

### 1. 动手跟着做

这门课是"做中学"。每个小项目都能真正跑起来，看到结果。所有命令都在**终端**里执行，你只管照着敲。

### 2. 记住一个核心比喻：黑板与课堂

我们会把 DORA 的原理，统统比作一间**有大黑板的教室**：

- **节点（Node）** = 教室里的同学，各做各的事；
- **数据流（Dataflow）** = 谁把什么写到黑板、谁去看，这套分工；
- **共享内存（Shared Memory）** = 那块大家都看得见的**黑板**本身。

只要抓住"黑板"这个画面，后面再抽象的概念都不难。

### 3. 每章都长一个样

每一章都按同样的结构组织：

- **学习目标**：这一章学完你能做什么；
- **正文**：配比喻、配图、配代码，一步步讲；
- **动手练习**：小任务 + 参考答案；
- **常见报错 FAQ**：把你可能踩的坑提前排掉；
- **小结**：回顾要点，衔接下一章。

### 4. 关于术语

第一次出现的专业词，我们都会用**中文（English）** 的方式标注，比如"节点（Node）"。之后就用中文。
