# 第三章 · 第一个数据流：dora 小飞机

> 本章主题：⚡ 第一个数据流

本章我们要运行一个现成的 DORA 项目——**dora 小飞机**。你不需要写任何代码，只需要执行命令就能：编译一个 Rust 项目、运行一条数据流、用键盘控制一架小飞机飞起来。

这是你第一次亲手感受"数据流"在节点之间流动——也是后续自己动手写节点的基础。

## 本章内容

- **[3.1 获取并编译小飞机工程](./build-aviation)**：克隆源码，编译三个二进制文件
- **[3.2 dora run 跑起来](./run-dataflow)**：启动数据流，用键盘控制小飞机
- **[3.3 读懂 dataflow.yml](./read-yaml)**：理解控制器与小飞机之间的"值日表"
- **[3.4 看懂流动的数据](./data-cmd-pose)**：了解 `cmd` 和 `pose` 的格式
- **[3.5 啊哈实验](./aha-experiment)**：删掉连线看看会发生什么

## 前置要求

- 已完成[第二章](../environment/)的环境搭建，能运行 `dora --version` 和 `dora --help`
- 安装了 git
- 机器有图形显示环境（非纯服务器），用于弹出 Bevy 窗口

:::tip 本章不用写代码
本章只需要在终端里执行命令、观察现象，建立对"数据流"的直观感受。从第四章起才需要动手写 Python 节点。
:::

:::warning 图形驱动注意
小飞机使用 Bevy 游戏引擎渲染窗口，需要你的机器支持图形驱动（Vulkan / Metal / OpenGL）。绝大多数台式机和笔记本的集成显卡都满足。**这不等于课程后面说的"AI 模型需要 GPU"**——图形驱动和 AI 计算是两回事，本章可以放心跑。
:::
