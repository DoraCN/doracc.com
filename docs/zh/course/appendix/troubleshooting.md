# 常见报错与求助指南

遇到报错别慌——这很正常，人人都会遇到。这一页教你**排错的思路**和**如何高效求助**。各章末尾还有针对该章的"常见报错 FAQ"，本页则是通用总纲，会随课程不断补充。

## 排错三步法

1. **看清报错的最后几行**：错误信息通常"最后一句"最关键，先读它，再往上看。
2. **定位是哪一类问题**（见下表），对症下药。
3. **缩小范围**：把"改动"一点点回退，看是哪一步引入的问题。

## 常见问题分类

| 现象 | 多半是 | 先试试 |
|------|--------|--------|
| 下载/安装很慢或失败 | 网络 / 镜像问题 | 确认走了朵拉魔盒的**内置国内镜像**；重试；换网络 |
| 找不到命令（command not found） | 环境没就绪 / 没在内置终端里 | 在**朵拉魔盒内置终端**执行；跑一次"环境自检" |
| 编译报错 | 工具链或依赖问题 | 看报错最后一句；必要时"重置引擎"重编 |
| 节点起不来 / 数据流跑不通 | YAML 写错 / 路径不对 | 用 `dora validate dataflow.yml` 校验；核对 `id`、`inputs`、`path` |
| 画面/窗口打不开 | 图形环境限制 | 确认不是纯无界面服务器；见第三章的说明 |
| 模型下载失败 | 模型源问题 | 确认走 ModelScope；重试；检查磁盘空间 |

## 善用这些"自查"工具

- `dora doctor`：一键系统诊断。
- `dora validate dataflow.yml`：检查数据流配置对不对。
- 朵拉魔盒的**环境自检**：检查工具链、镜像连通性、磁盘空间等。

## 如何高效求助

在 [DORA 中文社区](https://github.com/DoraCN) 提问时，附上这些信息能让别人更快帮到你：

1. **你的环境**：操作系统（Windows/macOS/Linux）、用的是不是朵拉魔盒。
2. **你在做什么**：第几章、哪一步、想达到什么。
3. **完整报错**：把报错**从头到尾**复制出来（别只截最后一行）。
4. **你已经试过什么**：这能避免别人重复建议。

:::tip 小提醒
把报错原文（英文也没关系）直接搜索一下，往往能很快找到答案。看不懂的英文，可对照[术语中英对照表](./glossary)。
:::
