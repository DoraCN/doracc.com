# 比喻体系：黑板与课堂（Metaphor System）

> 本课程讲解技术原理的**核心比喻**。全课统一使用，禁止临时另起炉灶。
> 原则：费曼学习法，假设听众是初中生；能用一块黑板讲清的，绝不引入术语。
> 与故事线（器官成长）配合使用：**器官比喻负责动机与代入感，黑板比喻负责讲清原理**。

## 一、核心画面（三句话讲清 DORA 为什么快）

- **DORA 的教室**：教室里有一块**超级大黑板**，所有同学同时看得见，想写的人直接上去写。**没人需要抄到自己本子上才能看懂，抬眼看就行。**
- **ROS2 的教室**：教室里**没有黑板**，同学之间只能**传纸条**——写→折→传→拆→誊抄→再折→再传，一路折拆抄。
- 结论：一个初中生听到"黑板 vs 传纸条"，无需任何术语，就能直觉理解 DORA 为什么比 ROS2 快得多。

## 二、全概念映射表（DORA 黑板 vs ROS2 传纸条）

| 概念 | DORA（黑板教室） | ROS2（传纸条教室） | 为什么 DORA 更好 |
|------|------------------|-------------------|-----------------|
| 节点（Node） | 教室里的同学，各有职责（读/算/写） | 也是同学 | — |
| 数据流（Dataflow） | 值日表/分工图（YAML）：谁读谁写 | 值日表要先"编译"成指令才能用 | YAML 像值日表般直觉；ROS2 改一次要重翻译 |
| 消息/数据 | 黑板上写的字、公式、图 | 纸条上的内容 | — |
| 输入（Input） | 抬头看黑板 | 接纸条、拆开、读 | — |
| 输出（Output） | 站起来在黑板上写 | 写纸条、折好、递出去 | — |
| 共享内存（shared memory） | **黑板本身**——共用、大家直接看得到 | **没有黑板** | DORA 本质就是"给 ROS2 加了一块黑板" |
| 零拷贝（zero-copy） | 不把黑板内容抄到本子再想，直接对着黑板做 | 每人必须拆纸条、誊到本子（序列化/反序列化）才处理 | 省掉"拆→抄→折"全部开销，是 10-17 倍差距的根源 |
| Arrow 统一格式 | 黑板有**统一书写规范**（中文+标准符号），人人入校就学 | 每人用自己的速记，接收方得先翻译 | DORA 统一"文字"，多语言直接读懂 |
| Topic（发布订阅） | 老师在讲台说"看黑板"，全班听到 | 给每人单独写纸条 | 同机靠黑板不出声；ROS2 同班也传纸条 |
| Service（一问一答） | 小明问小红"第三题答案？"，小红答"42" | 一来一回都要折传拆 | 同上 |
| Action（长任务边做边报） | 布置卷子，小华举手"做一半了""快了""交卷" | 每次进度都写一张纸条 | 同上 |
| Streaming（连续可打断） | 老师带读课文，可随时举手打断 | 连续传纸条 | 同上 |
| 协调器（Coordinator） | 班主任/老师，管理全班 | 管理角色 | — |
| 守护进程（Daemon） | 每排组长，管这一排 | 类似 | — |
| 节点仓库（Node Hub） | 学校代课老师库，缺谁调谁 | 自己找人培训再编入 | `hub: dora-yolo` 一行引入即用 |
| 录制回放 | 课堂录像，可回看/慢放/换人重看 | rosbag 可录但格式重 | 自带且支持节点替换回放 |
| 多机分布式 | 隔壁班另一块黑板，跨班靠广播/内线电话 | 跨班也传纸条 | DORA 自动"同班走黑板、跨班走电话"，学生不用管 |

## 三、Python 与 Rust（教室里的两种同学）

- **Python 同学**：灵活、学得快、能讲故事，但算术略慢——适合"理解画面""用 AI 决策"。
- **Rust 同学**：心算极快、从不写错、不太会聊天——适合"高频读写""每秒上千次处理"。
- 关键：**两人用同一套黑板书写规范（Arrow）**，Python 写的 Rust 直接读，无需互译。

## 四、费曼时刻（现场可演示的讲法）

1. **为什么传纸条慢**：现场演示"写→折→传→拆→读→誊→写回→折→传回"共 8 步；黑板只需"写一步、抬眼一步"。2 步 vs 8 步，数据越大差距越大。
2. **黑板会不会乱**：引出 YAML 值日表——分区、分工，各写各的区，不是乱写。
3. **都去写会不会挤**：写完就走、读的人不排队，说明共享内存不等于堵塞。
4. **两个班如何配合**：同班黑板 + 跨班电话，DORA 自动处理，学生只写 YAML。

## 五、与故事线（器官成长）的双层配合

| 器官故事（动机层） | 黑板课堂用语（原理层） |
|-------------------|----------------------|
| 给小多装神经（收发消息） | "小多走进教室，学会了读黑板上的字" |
| 给小多装眼睛（摄像头+YOLO） | "小多不仅会读，还能看懂黑板上的照片——里面有个苹果" |
| 给小多装大脑（VLM） | "小多看着照片想了想：这个苹果该放左边篮子" |
| 给小多装手（机械臂） | "小多走到黑板前，伸手抓住了苹果" |

> 双层零冲突：故事层讲"小多获得新能力"，原理层讲"这能力在黑板上如何表现"。

## 六、边界与诚实交代

- 黑板比喻覆盖约 90% 概念。**Arrow 的列式存储优势**黑板讲不透时，另用"像 Excel 按列找比逐行读快"补充，不硬套黑板。
- **多机分布式**用"隔壁班黑板 + 广播/电话"延伸，写作时明确这是同一比喻的扩展。

## 七、使用纪律

- 讲原理、画图、对比 ROS2 → 一律用黑板课堂比喻。
- 讲动机、章节开场、能力解锁 → 用器官成长故事。
- 首次引入某概念时，先给黑板画面，再给术语（中文括注英文），最后给代码。
