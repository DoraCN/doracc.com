## 1.2《为什么是 DORA？——和 ROS2 比一比》详细方案

### 一、讲解原则

1. **不假设学生懂 ROS2**：ROS2 只作"参照物"，帮助理解 DORA 为何这样设计，不做 ROS2 教学。
2. **不贬低 ROS2**：客观诚实，讲清各自所长；ROS2 不是"差"，而是"更老更全"。
3. **全程黑板比喻**：所有对比都落到"黑板 vs 传纸条"。
4. **数字有出处**：10-17 倍、~500µs vs 1-10ms 均来自官方，正文标注"官方数据，随版本可能变化"。

### 二、内容结构（单独成节，中等篇幅）

**① 开场：小多的烦恼（引出"中间件"）**
- 小多身上有眼睛、大脑、手，它们如何高效配合？→ 需要一套"班级管理制度"让部件协作，这套制度叫**机器人中间件（Middleware）**。
- 一句话交代 ROS/ROS2：最有名的老牌"制度"；DORA 是为 AI 时代新设计的一套。

**② 核心三问（每问一个黑板画面）**

| 问题 | 黑板讲法 | 硬结论 |
|------|---------|--------|
| 快不快？ | 黑板 vs 传纸条：现场演示"写折传拆誊"8 步 vs"写、抬眼"2 步 | 同机零拷贝，~500µs vs 1-10ms，比 ROS2 Python 快 10-17 倍 |
| 好不好学？ | YAML 值日表 vs 先把值日表编译成指令；`dora run` 一条命令开跑 | 用 YAML 描述数据流，改完即跑，无需代码生成 |
| 懂不懂 AI？ | 黑板统一书写规范（Arrow）+ Python 同学，直接把照片贴黑板 | 原生 Arrow + Python 优先，直接接 YOLO/大模型 |

**③ 一张总对比表（8 维度）**

| 维度 | DORA（黑板教室） | ROS2（传纸条教室） |
|------|-----------------|-------------------|
| 同机延迟 | ~500µs，共享内存零拷贝 | 1-10ms，需序列化 |
| 描述方式 | YAML 值日表，改完即跑 | 需代码生成/编译 |
| 启动 | `dora run` 一条命令 | 组件较多、步骤较繁 |
| 数据格式 | Apache Arrow，多语言零翻译 | 需消息定义/翻译 |
| 语言 | Python/Rust/C/C++ 自由混用 | 多语言（偏 C++/Python） |
| AI 接入 | 原生友好（视觉/大模型） | 偏传统机器人 |
| 附带能力 | 录制回放/容错/可观测内置 | 各有对应组件 |
| 生态成熟度 | 年轻，成长中 | 庞大成熟（导航/驱动/社区） |

**④ 诚实交代（建立信任，务必保留）**
- 类比：ROS2 像**运营多年的老牌名校**（资源多、规矩多、什么都有）；DORA 像**新式学校**（轻快、专注 AI、上手快）。
- 明确：选谁看场景；学会 DORA 的"黑板思想"，日后理解 ROS2 也更容易；DORA 还提供 ROS2 桥接，两者能协作。

**⑤ 小结一句话**
- **"DORA = 给机器人配了一块共享黑板。"** 记住这句，全课后面都能串起来。
