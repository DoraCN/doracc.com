# 朵拉魔盒 / DoraBox 功能规格

> 面向《跟朵拉学机器人：从零到具身智能》课程的一站式隔离环境桌面工具。
> 目标：让零基础学生"打开即用"，在完全隔离、零系统污染的环境中，从源码编译并运行 dora 1.0 RC。
> 本工具由作者开发；课程只讲"如何使用"，不讲其内部实现。本文件用于对齐工具能力，指导环境章写作。

## 一、产品定位

- **一句话**：一个装好了一切的"四次元口袋"——打开它，就有一套干净、隔离、可复现的 DORA 开发环境。
- **面向对象**：零基础学生（多数不熟悉命令行、环境变量、包管理）。
- **平台**：macOS、Windows、Linux 三平台一致体验。
- **当前阶段定位（重要）**：现阶段只做 **MVP —— "能把环境跑起来的工具"**。教学闭环、错误自救、离线整包、进度/证书等更高级的教学软件能力，见 `spec/dorabox-roadmap.md`，**暂不实现**。
- **核心承诺**：
  1. 零系统污染：不改动系统 Rust / Python / PATH / shell 配置。
  2. 零配置门槛：镜像、工具链、依赖开箱即用。
  3. 可复现：所有人得到完全一致的 dora 版本与运行结果。

## 二、核心设计原则

1. **完全隔离**：Rust 工具链与 Python 环境都安装在魔盒工作目录内，与系统互不影响。
2. **工作目录由用户指定**：三平台文件系统差异大，启动时让用户显式选择工作目录，不预设/不硬编码默认路径。
3. **一切自包含**：可执行程序、第三方包、编译临时文件、模型、课程代码等全部落在工作目录内，卸载=删目录。
4. **内置终端为唯一官方路径**：课程所有命令都在魔盒内置终端执行，环境变量由魔盒注入，所见即所得。
5. **国内镜像默认开启**：crates、pip、模型、git 全部走国内镜像，编译/下载不卡顿。
6. **版本锁定**：dora 源码锁定到指定 commit，保证教学与学生结果一致。
7. **可清理**：支持分级清理，一键彻底卸载不留残渣。

## 三、功能模块

### 1. 工作目录管理（启动第一步）

- **启动时选择工作目录**：首次启动引导用户显式选择一个工作目录（可自由放在任意磁盘/分区），不假设任何默认位置。
- **三平台适配**：
  - 提供目录选择器，校验路径可写、空间充足、无特殊字符风险。
  - Windows 注意盘符与长路径；macOS 注意权限与外置卷；Linux 注意挂载点与权限。
- **记忆与切换**：记住上次选择的工作目录；支持后续在设置中更换或维护多个工作目录。
- **自包含**：Rust 工具链、Python 环境、dora 源码与产物、缓存、课程工程全部集中在该工作目录下。

### 2. 环境隔离引擎（零污染核心）

- **一切归于工作目录**：所有内容集中存放于用户所选工作目录，卸载/清理时针对该目录即净。
- **Rust 工具链隔离**：
  - 自定义 `RUSTUP_HOME` / `CARGO_HOME` 指向工作目录，绝不写入系统 `~/.cargo`、`~/.rustup`。
  - 自动下载并安装指定版本 Rust 工具链（含 `rustc`、`cargo`、`rustup`）。
- **Python 环境隔离**：
  - 内置独立 Python 运行时 + 隔离虚拟环境（基于 uv 或等价方案），不使用/污染系统 Python。
  - 所有 `pip install` 均落在隔离环境内。
- **环境变量注入**：仅在魔盒内置终端会话内注入 `PATH`、`RUSTUP_HOME`、`CARGO_HOME`、`VIRTUAL_ENV` 等，进程级隔离，退出即失效。

### 3. 一键环境初始化

- **首次启动引导**：图形化向导，逐步完成环境准备（无需学生手动敲命令）。
- **准备步骤（自动化 + 进度可视化）**：
  1. 选择/确认工作目录。
  2. 检测平台与磁盘空间。
  3. 下载并安装隔离 Rust 工具链。
  4. 创建隔离 Python 环境。
  5. 配置国内镜像源。
  6. 拉取 dora 源码并编译。
  7. 环境自检，报告"就绪"。
- **断点续做**：某步失败可重试，不必从头再来。

### 4. 镜像源管理（内置国内镜像）

- **默认开启的镜像**：
  - Rust crates：rsproxy / 字节 crates 镜像（配置 `config/cargo-config.toml`）。
  - Python pip：清华 TUNA 源。
  - 模型权重：**优先 ModelScope（魔搭）**；ModelScope 无对应模型时回退 hf-mirror。
  - Git：AtomGit（dora 源码本身即托管于此，国内直连）。
- **可切换**：设置面板中可在"国内镜像 / 官方源"间切换（进阶用户）。
- **对学生透明**：默认无需任何配置即生效。

### 5. dora 源码获取与编译

- **锁定源码**：从 `https://atomgit.com/dora-rs/dora` 克隆，检出 commit `25bac6b3e5ed7435f49bd494e0ff4ef81ee0a674`。
- **一键编译**：调用隔离 cargo 编译 dora CLI 及相关组件。
- **产物管理**：编译出的 `dora` 二进制放入工作目录并纳入内置终端 `PATH`。
- **编译缓存**：保留编译产物，重复打开无需重编。
- **验证**：编译后自动执行 `dora --version` 等校验，确认可用。

### 6. 内置终端

- **完整交互终端**：基于 pty（如 xterm.js + 后端 pty），支持真实命令交互、颜色、补全。
- **环境自动就绪**：打开即处于隔离环境，`dora`、`cargo`、`python`、`pip` 均可直接调用；默认工作路径为用户所选工作目录。
- **跨平台一致**：屏蔽 macOS/Windows/Linux 的 shell 差异，课程命令三平台通用。
- **常用能力**：复制粘贴、字体缩放、清屏、多标签（可选）。

### 7. 工作区 / 课程工程管理

- **课程仓库集成**：一键克隆课程仓库（含 `examples/`）到工作目录，或指向本地已克隆目录。
- **章节导航**：列出各章示例工程，一键在内置终端切换到对应目录。
- **工作目录约定**：课程工程默认置于用户所选工作目录下，便于统一运行与清理。

### 8. 依赖与节点管理

- **Python 依赖**：在隔离环境内执行 `pip install`（走清华源）。
- **dora Node Hub**：支持课程用到的 `hub:` 节点拉取（走镜像/AtomGit）。
- **模型下载**：轻量 CPU 模型（如 YOLOv8n、小型 VLM）**优先经 ModelScope 下载**并缓存到工作目录，缺失时回退 hf-mirror。

### 9. dora 运行辅助（可选图形面板）

- **快捷操作**：图形按钮触发 `dora build` / `dora run` / 停止 / 查看日志（底层仍等价于内置终端命令，便于教学对照）。
- **日志查看**：结构化展示运行日志，便于零基础学生排错。
- **数据流状态**：显示当前运行的 dataflow、节点状态（对接 dora 可观测能力，进阶可选）。

### 10. 环境自检与诊断

- **一键体检**：检查工作目录可用性、Rust/Python/dora 版本、镜像连通性、磁盘空间、编译产物完整性。
- **常见问题提示**：针对典型报错给出中文修复建议（与课程 FAQ 呼应）。
- **版本信息面板**：清晰展示 dora commit、Rust 版本、Python 版本、当前工作目录，确保与课程一致。

### 11. 更新与维护

- **课程/源码版本管理**：跟随课程指定的 dora commit；作者更新时可平滑升级。
- **工具自更新**：魔盒自身版本更新提示。

### 12. 卸载与清理（零污染闭环）

- **分级清理**：见"工作目录结构规划 - 分级清理策略"。
- **一键彻底卸载**：删除整个工作目录即彻底移除，不在系统目录、shell 配置、注册表（Windows）留下痕迹。

## 四、工作目录结构规划

> 核心思路：按"生命周期/易变性"分层，让半永久大件（工具链）、可重建缓存、引擎源码与产物、高频接触的课程代码彼此隔离，从而支撑分级清理与清晰的环境变量注入。

### 4.1 设计原则

1. **一切自包含**：所有内容都在用户所选工作目录内，卸载=删目录。
2. **按易变性分区**：工具链（半永久大件）/ 可重建缓存 / 引擎源码与产物 / 课程代码与练习 / 魔盒元数据。
3. **环境变量精准映射**：每个子目录对应一个明确的 env。
4. **跨平台稳妥**：目录名全小写 ASCII、无空格，规避 Windows 长路径/盘符坑。

### 4.2 目录树

```
<用户所选工作目录>/                 # 例如 D:\DoraBox 或 /home/xx/dorabox
├── .dorabox/                      # ① 魔盒元数据（隐藏，学生勿动）
│   ├── config.json                #   工作目录/镜像开关/版本锁记录
│   ├── state.json                 #   环境就绪状态、断点续做进度
│   └── logs/                      #   魔盒操作日志、编译日志
│
├── toolchains/                    # ② 语言工具链（半永久大件）
│   ├── rust/
│   │   ├── rustup/                #   RUSTUP_HOME（工具链本体）
│   │   └── cargo/                 #   CARGO_HOME（bin/ + registry 依赖缓存 + git）
│   └── python/
│       ├── runtime/               #   独立 Python 运行时（uv 安装）
│       └── venv/                  #   隔离虚拟环境 VIRTUAL_ENV（第三方 pip 包在此）
│
├── engine/                        # ③ dora 引擎（锁定源码 + 编译）
│   └── dora/
│       ├── src/                   #   AtomGit 克隆源码 @ 25bac6b3…
│       ├── target/                #   编译临时文件 + 产物（CARGO_TARGET_DIR，体积大）
│       └── bin/                   #   最终 dora 可执行程序（纳入 PATH）
│
├── caches/                        # ④ 可重建缓存（"清理缓存"首选目标）
│   ├── pip/                       #   PIP_CACHE_DIR
│   ├── uv/                        #   UV_CACHE_DIR
│   └── nodehub/                   #   dora Node Hub 拉取缓存
│
├── models/                        # ⑤ 模型权重（大件、下载贵，独立管理，不随清缓存删除）
│   ├── modelscope/                #   MODELSCOPE_CACHE（优先）
│   └── hf/                        #   HF_HOME（回退）
│
├── config/                        # ⑥ 注入的镜像/工具配置
│   ├── cargo-config.toml          #   crates 镜像（rsproxy/字节）
│   └── pip.conf                   #   pip 清华源
│
└── course/                        # ⑦ 课程内容（学生高频接触）
    ├── repo/                      #   git clone 的课程仓库（docs/ + examples/）
    └── workspace/                 #   学生自由练习/自建工程
```

### 4.3 环境变量映射表（内置终端注入）

| 目标 | 环境变量 | 指向 |
|------|---------|------|
| Rust 工具链 | `RUSTUP_HOME` | `toolchains/rust/rustup` |
| Cargo 主目录+依赖缓存 | `CARGO_HOME` | `toolchains/rust/cargo` |
| 编译产物/临时 | `CARGO_TARGET_DIR` | `engine/dora/target` |
| Python 虚拟环境 | `VIRTUAL_ENV` | `toolchains/python/venv` |
| pip 缓存 | `PIP_CACHE_DIR` | `caches/pip` |
| uv 缓存/Python 安装 | `UV_CACHE_DIR` / `UV_PYTHON_INSTALL_DIR` | `caches/uv` / `toolchains/python` |
| 模型（魔搭优先） | `MODELSCOPE_CACHE` | `models/modelscope` |
| 模型（回退） | `HF_HOME` | `models/hf` |
| 可执行程序 | `PATH`（前置） | `cargo/bin` + `engine/dora/bin` + `venv/bin`（win 为 `Scripts`） |
| pip 镜像 | `PIP_CONFIG_FILE` | `config/pip.conf` |

> 第三方包落点：Rust 依赖缓存在 `CARGO_HOME/registry`（cargo 规范，不硬拆）；Python 第三方包装进 `venv`；两者都在工作目录内，仍满足"全部自包含"。

### 4.4 分级清理策略

| 操作 | 删除范围 | 效果 |
|------|---------|------|
| 清理缓存 | `caches/` + `engine/dora/target` | 释放最多空间，源码/工具链/模型保留，下次自动重建 |
| 清理模型 | `models/` | 学生显式操作，删除昂贵模型权重（下次用时重下） |
| 重置引擎 | `engine/` | 重新克隆+编译 dora，工具链保留 |
| 重置环境 | `toolchains/` + `engine/` + `caches/` + `models/` | 保留 `course/`（学生代码不丢） |
| 彻底卸载 | 整个工作目录 | 零残留 |

> 把"课程代码"单独放 `course/`、"可重建物"放 `caches/`、"昂贵模型"独立放 `models/`：清缓存释放空间时不误删模型，也绝不误删学生自己的代码。

### 4.5 跨平台注意

- **目录命名**：全小写、无空格、无中文，避免 Windows/编译工具链踩坑。
- **PATH 分隔与 bin 目录**：Windows 用 `;` 与 `Scripts/`，macOS/Linux 用 `:` 与 `bin/`，由魔盒按平台注入。
- **长路径**：Windows 建议工作目录尽量靠近盘符根（如 `D:\DoraBox`），减少 `target/` 深路径超长风险。
- **权限/外置卷**：macOS 外置卷、Linux 挂载点需校验可执行与可写权限。

## 五、典型用户流程

### 首次使用
1. 下载安装朵拉魔盒 → 打开。
2. **选择工作目录**（任意磁盘/分区）。
3. 点击"准备环境" → 向导自动完成（工具链、Python、镜像、编译 dora）。
4. 看到"环境就绪" → 一键克隆课程仓库到工作目录。
5. 在章节列表选择"小项目①" → 打开内置终端 → `dora build` / `dora run`。

### 日常使用
1. 打开魔盒（自动加载上次工作目录，环境已就绪）。
2. 选择章节工程 → 内置终端。
3. 跟随课程敲命令、跑数据流、看日志。

## 八、技术选型（工具实现）

> 本工具由作者开发，课程不讲其实现；此处仅作实现方向记录。

- **桌面框架**：**Tauri v2**（Rust 后端 + Web 前端）。
  - 与 dora "100% Rust" 品牌一致；三平台打包，体积/内存小。
  - Rust 后端天然擅长本工具的核心工作：下载/管理 Rust 工具链、克隆并编译 dora 源码、拼装环境变量、拉起子进程与下载（`std::process`、`reqwest` 等）。
- **内置终端**：前端 **xterm.js** + 后端 **portable-pty**（Rust）实现真实 pty 交互。
- **前端**：Web 技术栈（用于图形向导、终端界面、运行面板、诊断页）。
