# 朵拉魔盒 / DoraBox 功能路线图（Roadmap）

> 本文件记录朵拉魔盒的分阶段规划。
> **当前阶段只实现"能把环境跑起来"的 MVP（阶段 0）；本文件阶段 1 及以后的内容为规划，暂不实现。**
> 目的：把已讨论到的教学软件能力沉淀下来，避免遗忘；后续按优先级择机落地。
> 参考对象：Replit、JetBrains Academy / EduTools、Swift Playgrounds、Arduino IDE、mBlock、DataCamp、慕课类平台、LEGO / VEX 机器人教学软件的共性做法。

---

## 阶段 0：MVP —— "能跑起来的工具"（当前唯一实现范围）

定位：让零基础学生在完全隔离、零污染的环境中，一键从源码编译并运行 dora 1.0 RC。

已在 `spec/dorabox-features.md` 定义并纳入 MVP 的能力：

- 工作目录选择与管理（启动时用户显式指定）。
- 环境隔离引擎（Rust 工具链 + Python 环境隔离，进程级 env 注入）。
- 一键环境初始化向导（含断点续做）。
- 内置国内镜像（crates / pip / 模型 ModelScope 优先 / AtomGit）。
- dora 源码获取与编译（锁定 commit）。
- 内置终端（xterm.js + portable-pty，唯一官方路径）。
- 课程仓库克隆与章节目录切换。
- 依赖与节点管理、模型下载。
- 基础运行辅助（build/run/日志）、基础环境自检、更新、卸载。
- 六大目录结构与分级清理。
- 技术栈：Tauri v2。

**MVP 之外的一切，见下方阶段 1+，暂不实现。**

---

## 阶段 1：教学闭环（Learning Loop）

> 目标：从"能跑起来"升级到"学得下去"。这是与普通环境工具最大的区别。

### 1.1 学习进度与断点续学 〔高〕
- **描述**：记录每章 / 每个小项目的完成状态、当前进度、上次学习位置；主界面提供"继续学习"。
- **实现要点**：在 `.dorabox/` 下新增 `progress.json`（与 `state.json` 分离，前者学习进度、后者环境状态）。
- **参考**：慕课/DataCamp 的进度条与"继续上次"。

### 1.2 引导式教学：课程与终端并排 〔高〕
- **描述**：左侧课程步骤（来自课程仓库的结构化内容），右侧内置终端/运行面板；课文中的命令可"一键运行到终端"，逐步高亮引导。
- **实现要点**：解析课程仓库中章节的步骤元数据；命令块带"运行"按钮；步骤完成打勾。
- **参考**：Replit、Swift Playgrounds 的分栏引导。

### 1.3 练习自动校验与即时反馈 〔高〕
- **描述**：学生完成练习后自动判定是否达标，给出通过/未通过与提示。
- **与 dora 结合点**：善用 dora 的 **录制回放 `dora replay --assert`**——用预置的期望输出对学生的数据流做断言校验；或对节点输出做规则校验。
- **实现要点**：每个练习附带校验脚本/期望产物；结果以友好中文反馈；支持"查看提示""对照参考答案"。
- **参考**：JetBrains Academy 的任务校验、DataCamp 的 submission check。

### 1.4 首次运行新手引导（Tour） 〔中〕
- **描述**：首启时高亮讲解"终端、章节列表、准备环境、运行按钮"等关键区域。
- **实现要点**：可跳过、可重看；仅首次自动触发。

## 阶段 2：错误处理与求助（Beginner Safety Net）

> 目标：零基础学生"卡住时能自救"，降低劝退率。

### 2.1 友好错误翻译 + 一键修复 〔高〕
- **描述**：捕获常见编译/运行/网络报错，翻译成中文人话，给出成因与修复建议，并提供一键操作（如"重装工具链""切换镜像重试""清理并重编"）。
- **实现要点**：维护"报错特征 → 中文解释 + 修复动作"的规则库；与课程 FAQ 联动。
- **参考**：Arduino IDE / 商业 IDE 的错误提示增强。

### 2.2 一键导出诊断包 〔高〕
- **描述**：一键打包日志、环境信息、版本、工作目录结构快照，生成可发给老师/社区的求助文件（脱敏）。
- **实现要点**：收集 `.dorabox/logs/`、版本面板信息、镜像连通性结果；打包为 zip。
- **参考**：企业软件的"收集诊断信息"、VSCode 的 issue reporter。

### 2.3 一键重置 / 自愈 〔中〕
- **描述**：环境损坏时引导式恢复到干净可用状态，无需卸载重来。
- **实现要点**：复用分级清理（清缓存/重置引擎/重置环境），配引导式向导与风险提示（保护 `course/` 与 `models/`）。

## 阶段 3：网络与离线可靠性（Offline & Resilience）

> 目标：适配国内课堂/机房/弱网环境。

### 3.1 完全离线整包 〔高〕
- **描述**：提供含工具链 + dora 源码 + 依赖 + 模型的离线安装包，一次拷贝全班机器可用，无需联网。
- **实现要点**：可导出/导入"环境快照"或离线资源包；校验完整性。
- **参考**：LEGO / VEX 机器人教学软件对机房离线部署的重视。

### 3.2 断点续传 + 镜像自动回退 + 重试 〔高〕
- **描述**：大文件（依赖、模型、源码）下载中断可续；镜像失败自动换源；失败自动重试并给出进度。
- **实现要点**：下载层统一封装续传与多源回退（ModelScope→hf-mirror 等）。

### 3.3 安装前系统预检 〔高〕
- **描述**：安装/初始化前检测磁盘空间、内存、CPU 架构（Apple Silicon / x86）、Windows 长路径开关 / VC++ 运行库等，提前拦截，而非编译到一半失败。
- **实现要点**：预检报告 + 不满足项的修复指引。

---

## 阶段 4：内容与更新（Content & Update）

### 4.1 课程内容独立更新 〔高〕
- **描述**：课文/示例脱离 App 单独更新（拉取最新课程仓库），提示"课程有更新"并展示更新说明。
- **实现要点**：课程仓库版本检测与增量拉取；与 App 版本解耦。
- **背景**：dora RC 阶段变动频繁，课程需要能快速迭代。

### 4.2 磁盘占用可视化 〔中〕
- **描述**：按六大目录（工具链/引擎/缓存/模型/课程/元数据）展示各自占用，配合可视化清理入口。
- **实现要点**：目录体积统计 + 与分级清理联动。
- **参考**：商业软件的"存储管理"页。

### 4.3 多课程 / 多环境预留 〔中〕
- **描述**：架构上预留"课程"维度，未来可承载不止一门课。
- **实现要点**：工作目录下 `course/` 支持多课程；环境可按课程隔离或共享。

---

## 阶段 5：体验 / 合规 / 激励（Experience, Compliance, Motivation）

### 5.1 隐私与未成年人合规 〔高〕
- **描述**：受众含学生（可能未成年）：默认不采集数据、遥测严格 opt-in、本地优先、提供清晰隐私说明。
- **实现要点**：任何数据上报前显式同意；默认全本地。

### 5.2 成就 / 结业证书 / 作品导出 〔中〕
- **描述**：完成小项目给徽章；毕业项目发结业证书；可导出作品（抓取 demo 录屏 / `.drec` 回放）。
- **实现要点**：进度系统联动；证书生成；作品导出与分享。
- **参考**：Codecademy/慕课的证书与成就体系。

### 5.3 无障碍与偏好 〔中〕
- **描述**：深浅色主题、字体缩放、终端配色；UI 多语言预留。

### 5.4 匿名使用分析 + 崩溃上报（opt-in） 〔中〕
- **描述**：在用户同意前提下收集"卡在哪一步"等匿名数据，帮助作者优化课程与工具。
- **实现要点**：默认关闭；数据脱敏；可随时关闭。

---

## 阶段 6：后置能力（Future / 进校场景）

### 6.1 教师端 / 班级管理 〔后置〕
- **描述**：教师仪表盘、班级名册、作业分发与回收、学生进度监控、批量部署。
- **触发条件**：课程进入学校/机构规模化使用时再考虑。

### 6.2 作品展示画廊 / 社区互动 〔后置〕
- **描述**：学生作品展示、点赞、社区互助入口（Discord/社区站）。

---

## 优先级速览

| 优先级 | 能力 |
|--------|------|
| 高（阶段 1-4 核心） | 进度与续学、引导式并排教学、练习自动校验、错误翻译+一键修复、诊断包导出、离线整包、断点续传+回退、系统预检、课程独立更新、隐私合规 |
| 中 | 新手 Tour、自愈重置、磁盘占用可视化、多课程预留、结业证书/作品导出、无障碍、匿名分析 |
| 后置 | 教师端班级管理、作品画廊/社区互动 |

> 落地顺序建议：阶段 1（教学闭环）→ 阶段 2（错误自救）→ 阶段 3（离线可靠）→ 阶段 4（内容更新）→ 阶段 5（体验合规激励）→ 阶段 6（进校）。
