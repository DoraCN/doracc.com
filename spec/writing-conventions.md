# 写作约定（Writing Conventions）

> 保证全课程风格统一、体验一致。所有章节写作必须遵循。

## 1. 语言与语气

- 全中文正文，代码注释也用中文。
- 面向零基础：多用比喻与图解，避免未解释的术语；首次出现的术语给出通俗解释。
- 语气亲切、鼓励，减少劝退感。
- **叙述人设**：小多是"会说话"的机器人主角，可用第一人称旁白与学生互动（如"小多：我现在还看不见世界呢～"），文风活泼亲切。
- **核心比喻**：技术原理统一使用"黑板与课堂"比喻，详见 `spec/metaphor-system.md`；讲原理/对比 ROS2 用黑板，讲动机/能力解锁用"器官成长"故事，二者分层配合。

## 2. 术语约定

- **中文为主，首次出现括注英文**，如"节点（Node）""数据流（Dataflow）"；此后正文用中文。
- 全课统一术语，以 `spec/glossary.md` 术语表为准，不得同义混用。

## 2. 统一章节模板

每章按以下结构组织：

1. **学习目标**：本章学完能做什么（2-4 条）。
2. **前置要求**：需要先完成的章节/已具备的知识。
3. **正文**：分小节讲解，配代码与图。
4. **动手练习**：可运行的小任务，配预期结果，并附**可折叠的参考答案/题解**。
5. **常见报错 FAQ**：列出学生易踩的坑与解法。
6. **小结**：回顾要点，衔接下一章。

## 3. 故事线（贯穿全课）

- **一条主线**：所有小项目不是孤立 demo，而是像搭积木一样**逐步组装同一个小多**——每章给它增加一个"器官/能力"，最后合体为毕业项目。
- 机器人名称：**小多**（取 DORA 谐音"朵拉"的"多"，是学生的机器人小伙伴）。
- **名字分工**：朵拉（DORA）指要学的技术，不拟人化；小多是唯一机器人主角。开篇需点破"小多之名源自朵拉"的关系，避免读者误以为朵拉是机器人。
- **能力成长线（每项目解锁一项）**：
  1. 能收发（神经/信息传递）→ 小项目①②
  2. 能看见（眼睛/视觉感知）→ 小项目③
  3. 能协作/加速（Rust 高性能）→ 小项目④
  4. 能动作（手/移动，仿真控制）→ 小项目⑤
  5. 能思考并行动（大脑/VLM，语言→感知→决策→动作）→ 小项目⑥毕业项目
- 配套工具：**朵拉魔盒 / DoraBox**，正文提到隔离环境准备时统一使用此名。
- 因是主线递进，章节之间存在依赖，写作时需在"前置要求"中标明。

## 4. 环境与命令约定

- 所有命令默认在**桌面工具的内置终端**中执行；正文以此为默认语境。
- 涉及安装/下载的命令，默认走**内置国内镜像**，无需学生额外配置。
- 三平台差异（macOS / Windows / Linux）用 Rspress 的 **Tabs 组件**分开展示。
- 不引导学生改动系统全局环境（不碰系统 Python / 全局 PATH）。

## 5. 代码与示例

- 每章的可运行工程放在 `examples/<章节标识>/`。
- 正文代码片段要与 `examples/` 中的真实工程一致，可直接运行。
- dora 相关代码必须实测通过（RC 阶段，避免踩坑）后再写入。
- `dataflow.yml`、Python、Rust 代码均配中文注释。

## 6. 图示

- 数据流/架构图优先用 **Mermaid**（已接入 `rspress-plugin-mermaid`，客户端渲染），便于维护。
- 用法：\`\`\`mermaid 代码块。复杂流程也可辅以表格/有序步骤列表。

## 7. 文档站呈现约定

### 7.1 页面粒度
- **每个小节（如 1.1、1.2）一个独立页面**，通过 Rspress 侧边栏导航；不做超长单页。
- 页面文件与小节对应，`_meta.json` 控制顺序与标题。
- **目录 slug 用语义化英文、不带数字前缀**（如 `concepts`、`first-dataflow`，而非 `m01-concepts`），让 URL 美观；章节**顺序由 `_meta.json` 显式控制**，与目录名无关。

### 7.2 提示框（Callout）约定
统一使用 Rspress 容器，全课不得混用其他风格：

| 用途 | 语法 | 说明 |
|------|------|------|
| 小多旁白 | `:::info` | 标题写"小多说"，第一人称、活泼 |
| 普通提示 | `:::tip` | 小技巧、便利做法 |
| 踩坑/注意 | `:::warning` | 易错点、危险操作、平台差异警示 |
| 进阶延伸（可跳过） | `:::details` | 折叠，零基础可略过的深入内容 |

> ⚠️ Rspress 坑：当 callout 内含**列表**时，列表与结尾 `:::` 之间**必须留一个空行**，否则会导致紧随其后的下一个 callout 渲染失败。

### 7.3 命令与多平台
- 包管理命令用 `PackageManagerTabs`（Bun 优先）。
- 三平台差异命令用 `Tabs`/`Tab`（标签：macOS / Windows / Linux）。
- 命令默认在朵拉魔盒内置终端执行、默认走内置镜像。

## 8. 示例工程（examples/）约定

- 每个小项目一个目录，命名：`examples/m<章号>-<项目短名>/`，如 `examples/m2-hello-dataflow/`。
- 每个目录含：
  - `README.md`（中文）：项目目标、运行步骤、预期结果。
  - `dataflow.yml` 及各节点源码（Python/Rust），均配中文注释。
- 目录与章节小项目一一对应，正文代码片段须与目录内真实工程一致、可直接运行。
